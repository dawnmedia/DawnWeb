Use this checklist before kickoff to reduce back-and-forth and keep the project focused.

## Project Preparation

<CardGrid>
  <Card title="Scope and Objectives">
    - Define the pages, features, and success metrics
    - Choose the primary call to action
  </Card>
  <Card title="Content and Assets">
    - Gather copy, images, logo files, brand colors, and social links
    - Identify content that still needs writing or approval
  </Card>
  <Card title="Launch Requirements">
    - Confirm domain, hosting, analytics, form destinations, and integrations
    - List any legal, privacy, or accessibility requirements
  </Card>
</CardGrid>

## Project Execution

<Steps>
1. Align on goals, audience, sitemap, and project timeline.

2. Review design direction and approve the first page or core interface before expanding.

3. Test core flows, metadata, responsiveness, and contact paths before launch.
</Steps>
